Radiation v 0.1.1: Emergency Bug Fix Edition

Tonight I found out that I had a mistake in the strftime code when creating new posts. ~Erik was nice enough to help and now v 0.1.1 on GitHub seems to work

The problem was in the post.rb file at approximately line 23. I used %e for the day instead of %d. %e uses a leading space for days less than 9, while %d does use the leading zero. As a result, new posts created between November 1st and November 9th would have had an unexpected space in their file names, which caused at least one problem with the date parsing upon blog publishing.

At ~erik's recommendation, I just changed the %e to %d, and pushed v 0.1.1 to GitHub.

Should be good now if you just pull down the latest changes from the repo on Github. Really sorry for the trouble the bug may have caused!
